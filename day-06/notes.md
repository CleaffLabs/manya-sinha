# Day 06 - Error Handling and Executions

## Notes
* n8n stores workflow execution history in the execution log.
* Execution logs can be used to inspect successful and failed workflow runs.
* Error workflows help notify users when a workflow fails.
* Error Trigger node starts an error workflow.
* Stop and Error node can be used to intentionally raise an error.
* Error messages help identify and debug workflow issues.
* Workflow errors can occur due to invalid input data, configuration issues, or external service failures.
* Execution history is useful for debugging and fixing workflows.
