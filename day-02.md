# Day 02-APIs and Webhooks

## Notes
* API (Application Programming Interface) allows applications to communicate with each other.
* APIs use requests and responses to exchange data.
* Main components of an HTTP request:
  * URL
  * Method
  * Header
  * Body
* Common HTTP methods:
  * GET (retrieve data)
  * POST (send data)
* Credentials are used for authentication and authorization.
* Common authentication methods:
  * API Key
  * OAuth
* Main components of an HTTP response:
  * Status Code
  * Header
  * Body
* Common status codes:
  * 200 → Success
  * 401 → Unauthorized
  * 404 → Not Found
  * 500 → Server Error
* Webhooks are used to receive information when an event occurs.
* Webhooks act as "reverse APIs" and can trigger workflows automatically.
* Polling repeatedly checks for updates, while webhooks send updates instantly.
