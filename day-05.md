# Day 05 - Workflow Concepts and Useful Nodes

## Notes
* Workflows start with a trigger node.
* Workflows must be activated to run automatically.
* Multiple triggers can be used in a single workflow.
* Branching allows workflows to follow different paths based on conditions.
* IF nodes create conditional branches (True/False paths).
* Multiple output connections can duplicate items across paths.
* Edit Fields (Set) node is used to clean, format, and manage data.
* Aggregate node combines data from multiple items into a single output.
* Webhook node receives data from external applications and can trigger workflows.
* Slack node can be used to send notifications and messages.
* Data can be aggregated into a single summary message instead of sending multiple messages.
* Webhooks can be used to handle different events and trigger different workflow branches.
